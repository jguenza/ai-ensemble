AI-Ensemble

A prompt-engineered system combining structured knowledge embedding with a compiler-style solution generator.

Overview

AI-Ensemble is a structured reasoning system for problem-solving.

Instead of interacting with a generic chatbot, users work through a fixed reasoning pipeline that separates problem specification, alternative reasoning, and solution.

The system is available both as an online and as a fully local, low-carbon (“Green AI”) version that runs entirely on the user’s own machine.

Both versions share the same core principles: controlled reasoning, response variation, and explicit knowledge inputs.

The AI Pipeline

At the core of AI-Ensemble is a fixed three-stage pipeline designed to guide the user toward the optimal solution.

Scenario — prompt specification

This stage focuses on precisely defining the prompt to be used, including context, constraints, assumptions, and objectives.

Prompts may involve tasks such as:

  • algorithm creation
  • planning an experiment
  • application building
  • school project/studying

Poorly specified prompts are improved.

Ensemble — alternative reasoning

This stage generates multiple alternative answers or reasoning paths for the same prompt.

Its purpose is to expose uncertainty and variation conditional on the prompt before any synthesis or decision is made.

Compiler — grounded synthesis

This stage combines alternatives answers into a solution.

It applies curated knowledge and information mixtures to produce a defensible result.

Our unique value

Most AI systems merge all reasoning into a single response. AI-Ensemble deliberately separates stages of reasoning.

This separation allows us to deliver our curated knowledge to each stage. We expose answer variation to the user by multiple local models or reasoning paths online.

The stages work together when users move knowledge through the stages of our pipeline.

Pipeline way of working with AI

Users optimize their prompts. They benefit from expert prior information. And exploit answer variation given a prompt.

Knowledge is not only model based. AI-Ensemble operates on private, expert-provided knowledge sources.

Local “Green AI” Version

The local version runs entirely on the user’s own machine and is designed for efficiency, privacy, and independence.

  • CPU-only (no GPU required)
  • Runs on commodity hardware/testing is needed
  • Uses quantized models
  • No cloud dependency or external data access

In the local version, the Ensemble stage can be implemented using multiple small, quantized models running entirely on the user’s machine.

Design Philosophy

AI-Ensemble is built around a simple principle:

Reliable reasoning emerges from structure, curated, and knowledge diversity — not from single sources only.
