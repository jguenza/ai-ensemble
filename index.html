AI-Ensemble

AI-Ensemble is a role-playing learning game for expert reasoning. It is available as an online service and as a fully local system built on the same core logic.

Users do not interact with AI-Ensemble as a generic chatbot. Instead, they enter a designed role-playing environment in which learning happens through reasoning, decision-making, and confrontation with curated knowledge.

The role-playing learning game

At the core of AI-Ensemble is a role-playing learning game. Users take on professional roles such as:

  • coder
  • statistician
  • breeder
  • director or decision-maker

Each role frames how problems are presented and which trade-offs matter. The same underlying knowledge may lead to different conclusions depending on the role being played.

Learning is driven by engineered progression: knowledge is introduced deliberately, users must justify claims using available material, and progress can be blocked when understanding is insufficient.

Speed and fluency do not matter. What matters is reasoning, consistency, and evidence.

One system, two ways to run it

AI-Ensemble is offered in two distributions of the same system: an online version and a local version. Both share the same learning logic, grounding rules, and role-playing mechanics.

Online version

The online version provides the full role-playing learning game as a hosted service. It is intended for users who want immediate access and a managed environment without local setup.

Local version (CPU-only, low-carbon)

The local version is a direct spin-off of the online system. It runs entirely on the user’s own machine using quantized, CPU-only models.

  • No GPU required
  • No dependence on large cloud providers
  • User-configurable knowledge and settings
  • Offline-capable

The user can purchase either the online or the local version, depending on their needs. The learning game remains the same; only the deployment differs.

Reliability through controlled inputs

Across both online and local versions, AI-Ensemble follows a strict rule:

The system reasons only over explicitly provided and curated knowledge.

If information is not present, the system does not invent it. “I don’t know” is a valid and intentional outcome. Reliability comes from input control and explicit evidence, not from model scale.

Responsible AI by design

By offering a local, CPU-only version built on quantized models, AI-Ensemble reduces environmental impact and avoids dependence on GPU-heavy infrastructure and mega cloud providers.

This design supports privacy, sustainability, and long-term educational and professional use.
